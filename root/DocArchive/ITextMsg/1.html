For usage as part of TagTrade.org's portal, the PMwiki cookbook recipe
SelectQuery needs to be ported.

The problem:

  SelectQuery 1.0 only works with a MySQL database.  At TagTrade.org,
  however, we will likely set up a PostgreSQL database, part of which
  should be accessible through PMwiki.

The task:

  Write a variant of SelectQuery version 1.0 (or higher) which
  interfaces with PostgreSQL version 7.3.

Info about the recipe:

  http://www.pmwiki.org/wiki/Cookbook/SelectQuery

Conditions:

  The first one to accomplish the task will receive 20 EUR.  In case he
  follows the process outlined in the URN X-trade a.0.1 specifications
  (see below), then he'll receive an additional 10 EUR.  Following the
  process is strongly encouraged - guidance will be provided by me, upon
  request.

  Payment will be through bank transfer, Paypal, or whatever else is
  convenient for the person who provided the first solution and for me.
